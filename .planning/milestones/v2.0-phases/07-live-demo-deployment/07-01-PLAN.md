Create `deploy/` directory and two files:

**deploy/honeyprompt.service** — systemd unit per D-02. Key details:

- `[Unit]` section: `Description=HoneyPrompt honeypot server`, `After=network-online.target`, `Wants=network-online.target`
- `[Service]` section:
  - `Type=simple`
  - `User=honeyprompt`, `Group=honeyprompt` (dedicated unprivileged system user)
  - `WorkingDirectory=/var/lib/honeyprompt`
  - `ExecStart=/usr/local/bin/honeyprompt serve /var/lib/honeyprompt/landing` — landing dir is under /var/lib/honeyprompt so the derived DB path (`/var/lib/honeyprompt/landing/.honeyprompt/events.db`) stays within the persistent data directory, satisfying D-04
  - `Restart=always` per DEPLOY-03
  - `RestartSec=5`
  - `StartLimitIntervalSec=600`, `StartLimitBurst=5` — prevent restart storms
  - `KillSignal=SIGINT` — CRITICAL: the server's `shutdown_signal()` listens for SIGINT (Ctrl+C), not SIGTERM. Without this, `systemctl stop` sends SIGTERM which bypasses graceful shutdown and may lose in-flight events from the mpsc channel
  - `TimeoutStopSec=10` — give graceful shutdown time to flush DB writes
  - Sandbox hardening: `NoNewPrivileges=yes`, `PrivateTmp=yes`, `ProtectSystem=strict`, `ReadWritePaths=/var/lib/honeyprompt`
- `[Install]` section: `WantedBy=multi-user.target`

**deploy/Caddyfile** — Caddy reverse proxy per D-03:

```
honeyprompt.sh {
    reverse_proxy localhost:8080
}
```

That is the complete file. Caddy auto-provisions Let's Encrypt TLS. No additional config needed.

  <action>
Create `Dockerfile` at repo root per D-06 (local testing and future container deployment).

Multi-stage build:

**Stage 1 (builder):**
- `FROM rust:1.87-slim AS builder`
- `WORKDIR /app`
- `COPY . .`
- Install musl-tools: `RUN apt-get update && apt-get install -y musl-tools`
- Add target: `rustup target add x86_64-unknown-linux-musl`
- Build: `RUSTFLAGS="-C target-feature=+crt-static" cargo build --release --target x86_64-unknown-linux-musl`
- Strip binary: `strip target/x86_64-unknown-linux-musl/release/honeyprompt`

**Stage 2 (runtime):**
- `FROM gcr.io/distroless/static-debian12` — NOT `scratch` (Pitfall 5: scratch has no CA certs, no dynamic linker fallback)
- `COPY --from=builder /app/target/x86_64-unknown-linux-musl/release/honeyprompt /usr/local/bin/honeyprompt`
- `ENTRYPOINT ["/usr/local/bin/honeyprompt"]`

Do NOT add `CMD` — the user passes subcommand and args at runtime (e.g., `docker run honeyprompt serve /data`).
