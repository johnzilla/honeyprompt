    1. `pub fn build_config_from_inputs(domain: &str, bind_address: &str, tiers: Vec<u8>, page_title: &str) -> Config` -- pure function, constructs Config with callback_base_url set to `format!("https://{}", domain)` and theme="default". This is the unit-testable core.

    2. `pub fn check_dns(domain: &str) -> anyhow::Result<bool>` -- calls `std::net::ToSocketAddrs::to_socket_addrs` on `(domain, 443)`. Returns Ok(true) if resolves, Ok(false) if resolution fails. Does NOT error on resolution failure (per SETUP-02: non-blocking warning).

    3. `pub fn validate_and_write_config(config: &Config, path: &std::path::Path) -> anyhow::Result<()>` -- serializes Config to pretty TOML via `toml::to_string_pretty`, writes to path. On write failure, returns Err with message "Cannot write config: {path}: {error}" (per SETUP-03).

    4. `pub fn run_setup(path: &std::path::Path) -> anyhow::Result<()>` -- the interactive entry point. Uses dialoguer:
       - `Input::<String>::new().with_prompt("Domain").interact_text()?` for domain
       - `Input::<String>::new().with_prompt("Bind address").default("0.0.0.0:8080".into()).interact_text()?` for bind
       - `MultiSelect::new().with_prompt("Tiers to enable").items(&["Tier 1 (visible)", "Tier 2 (hidden)", "Tier 3 (meta)"]).defaults(&[true, true, true]).interact()?` for tiers (map selected indices to 1-indexed u8 vec)
       - `Input::<String>::new().with_prompt("Page title").default("Security Research Canary".into()).interact_text()?` for title
       - Calls check_dns, prints yellow warning via `eprintln!("Warning: DNS lookup for '{}' failed — callback URLs may not work until DNS is configured", domain)` if false
       - Calls build_config_from_inputs, then validate_and_write_config
       - On success prints "Wrote {path}/honeyprompt.toml -- run `honeyprompt generate` next."
