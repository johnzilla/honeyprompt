# Phase 3 — UI Design Contract: TUI Monitor

> Visual and interaction contract for the `honeyprompt monitor` TUI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Note:** This phase uses Ratatui (terminal UI), not a web component library.
> All design tokens map to terminal cell units, ANSI color pairs, and widget
> composition — not CSS pixels or shadcn presets.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Ratatui / crossterm) |
| Preset | not applicable |
| Component library | ratatui 0.29+ widgets (Table, Block, Paragraph, Gauge, Tabs) |
| Icon library | none — Unicode box-drawing chars + ASCII glyphs only |
| Font | terminal default (monospace) |

Registry safety gate: not applicable — no third-party component registries involved.

## Layout

Source: CONTEXT.md D-01, D-02 (locked decisions)

```
┌─────────────────────────────────────────────────────────────────────┐
│ STATS HEADER (3 rows fixed)                                         │
│  Detections: N   Sessions: N   T1: N   T2: N   T3: N   Replays: N  │
│  Status: [integrated | attached]   Listening: 0.0.0.0:PORT         │
├─────────────────────────────────────────────────────────────────────┤
│ FILTER BAR (1 row fixed)                                            │
│  Filter: [All | T1 | T2 | T3]   Sort: [time | tier | source]       │
│  Show replays: [off | on]                                           │
├─────────────────────────────────────────────────────────────────────┤
│ EVENT TABLE (fills remaining terminal height, scrollable)           │
│  TIME       TIER  CLASS         SOURCE IP   UA (truncated)  NONCE  │
│  SESSION    FIRES  REPLAY                                           │
│  ...rows...                                                         │
├─────────────────────────────────────────────────────────────────────┤
│ KEY HINT BAR (1 row fixed, bottom)                                  │
│  j/k scroll  tab filter  s sort  r replays  : cmd  q quit          │
└─────────────────────────────────────────────────────────────────────┘
```

**Row allocation (terminal lines):**
- Stats header: 3 lines (including Block border top/bottom + 1 content row)
- Filter bar: 3 lines (including Block border + 1 content row)
- Event table: terminal height minus 7 lines (fills remainder)
- Key hint bar: 1 line (no border, plain Paragraph)

**Column widths for event table (terminal cells):**

| Column | Width | Notes |
|--------|-------|-------|
| TIME | 12 | HH:MM:SS.mmm format |
| TIER | 4 | "T1 " / "T2 " / "T3 " |
| CLASS | 12 | "agent" / "crawler" / "unknown" — truncated |
| SOURCE IP | 16 | IPv4 or truncated IPv6 |
| UA | fills remaining − 52 | truncated with "…" |
| NONCE | 10 | first 8 hex chars + "…" |
| SESS | 10 | first 8 hex chars + "…" |
| FIRES | 5 | right-aligned |
| REPLAY | 6 | " [R] " or "      " |

Minimum terminal width: 80 columns. Below 80: collapse SESS and FIRES columns.
Minimum terminal height: 20 lines.

## Spacing Scale

Terminal UIs use cell units, not pixels. The 8-point scale maps to cell multiples:

| Token | Terminal Cells | Usage |
|-------|---------------|-------|
| xs | 1 cell | Icon gaps, inline padding between label and value |
| sm | 2 cells | Column separator padding |
| md | 4 cells | Block inner margin (left/right) |
| lg | 6 cells | Section separation (header blocks) |
| xl | 8 cells | Not used in this phase |

Block borders: all primary panels use `Borders::ALL` with a single-line border
style. Inner padding: left=1, right=1, top=0, bottom=0 (Ratatui `Padding`).

Exceptions: key hint bar — no border, no padding. Full-width raw Paragraph.

## Typography

Terminal monospace — no size variation possible. Differentiation through weight
(bold), color, and style (dim, italic where crossterm supports it).

| Role | Style | Weight | Usage |
|------|-------|--------|-------|
| Header label | normal | bold | Stats bar counter labels: "Detections:", "T1:" |
| Header value | normal | bold + accent color | Live counter numbers |
| Table header | normal | bold + underline | Column header row in event table |
| Table row | normal | normal | Event data rows |
| Table row selected | normal | bold + inverted bg | Currently highlighted row |
| Replay row | dim | normal | Rows where is_replay=true when shown |
| Key hint | dim | normal | Bottom key hint bar |
| Error / status | normal | bold + red | Error messages in stats header |

Line height: 1 (terminal standard — no configurable line spacing).

## Color

Source: CONTEXT.md "Color scheme and visual styling" — delegated to Claude's
discretion. Color palette chosen to read well on dark terminals (the default
for security researchers) with graceful fallback to 16-color terminals.

**Palette (dark terminal base — ANSI 256-color and RGB where supported):**

| Role | ANSI / RGB | Terminal fallback | Usage |
|------|-----------|-------------------|-------|
| Dominant (60%) | Default bg (terminal default) | Black | Main surface — all panels, table rows |
| Secondary (30%) | Dark gray #1e1e2e / ANSI 236 | DarkGray | Block borders, column separators, filter bar bg |
| Accent (10%) | Cyan #00d7ff / ANSI 51 | Cyan | Reserved for: active filter label, selected row highlight border, T1 tier badge |
| Success | Green #00af5f / ANSI 35 | Green | T2 tier badge, detection count value, "agent" classification label |
| Warning | Yellow #ffd700 / ANSI 220 | Yellow | T3 tier badge |
| Destructive | Red #ff5f5f / ANSI 203 | Red | "crawler" classification label when suppressed, replay [R] badge, error messages |
| Muted | DarkGray #626262 / ANSI 241 | DarkGray | Replay rows (dim treatment), key hint bar text, disabled filter labels |

Accent reserved for: active/selected filter tab label, highlighted row border,
T1 tier badge text. Never applied as a background fill.

60/30/10 mapping:
- 60% dominant: terminal default background — all panel bodies, table cell backgrounds
- 30% secondary: #1e1e2e block borders, header bar background tint, filter bar
- 10% accent: cyan highlights on exactly the 3 reserved elements listed above

**Tier color map (used in TIER column and stats header counts):**

| Tier | Color | Meaning |
|------|-------|---------|
| T1 | Cyan (accent) | Arbitrary callback — agent made outbound request |
| T2 | Green (success) | Conditional branch — agent evaluated logic |
| T3 | Yellow (warning) | Computed callback — agent performed computation |

**Classification color map (used in CLASS column):**

| Classification | Color |
|---------------|-------|
| KnownAgent | Green (success) — compliance detection |
| KnownCrawler | Red (destructive) — suppressed from count |
| Unknown | White (default) |

## Copywriting Contract

Source: REQUIREMENTS.md TUI-01, TUI-02, CONTEXT.md D-05, D-06, D-07.

| Element | Copy |
|---------|------|
| Primary CTA (key hint) | `q` quit |
| Stats header label | `Detections: {N}  Sessions: {N}  T1: {N}  T2: {N}  T3: {N}` |
| Replay indicator in header | `{N} replays hidden` (when replays=off) or `{N} replays shown` (when replays=on) |
| Replay toggle hint | `r toggle replays` |
| Empty state — no events yet | `Waiting for callbacks… Start the honeypot and trigger payloads to see events here.` |
| Empty state — filter active with no matches | `No events match the current filter. Press tab to change filter or r to show replays.` |
| Filter bar labels | `Filter: All | T1 | T2 | T3   Sort: time | tier | source` |
| Integrated mode status | `Serving on 0.0.0.0:{port}` |
| Attach mode status | `Attached to db: {path}` |
| Error — db not found | `Error: database not found at {path}. Run \`honeyprompt serve\` first.` |
| Error — server bind fail | `Error: port {port} already in use. Use --port to specify another.` |
| Destructive actions | None in this phase — quit (`q`) requires no confirmation |
| Command prompt | `:` opens inline command input — supported verbs: `quit`, `filter <tier>`, `sort <field>` |

No destructive confirmation dialogs required for Phase 3.

## Interaction Contract

Source: CONTEXT.md D-03, D-04, D-05 (locked). Key bindings beyond j/k/tab/:
delegated to Claude's discretion.

### Key Bindings

| Key | Action |
|-----|--------|
| `j` / `Down` | Scroll event table down one row |
| `k` / `Up` | Scroll event table up one row |
| `PgDn` | Scroll event table down one page |
| `PgUp` | Scroll event table up one page |
| `g` | Jump to top of event table |
| `G` | Jump to bottom (latest event) |
| `Tab` | Cycle active filter: All → T1 → T2 → T3 → All |
| `s` | Cycle sort field: time → tier → source → time |
| `r` | Toggle replay row visibility |
| `:` | Open inline command input (vim-style command line) |
| `q` / `Ctrl-C` | Quit monitor (no confirmation required) |
| `?` | Toggle key binding help overlay |

### Filter and Sort State

- Active filter and sort displayed in filter bar at all times.
- Filter and sort state is in-memory only — not persisted between invocations.
- Default on launch: filter=All, sort=time (newest first), replays=hidden.

### Event Table Behavior

- New events append to bottom of full event list. Table auto-scrolls to bottom
  if user is already at the bottom (last row selected or no selection).
- If user has scrolled up (not at bottom), new events do not force scroll — a
  "N new events" indicator appears in the filter bar area.
- Selected row highlights in inverted colors (accent border on block).
- Replay rows rendered in dim style when visible; hidden by default (D-05).
- Stats header counters exclude replay events from detection/session counts (D-07).

### Command Mode (`:`)

- `:` enters command input at bottom of screen, overlaying the key hint bar.
- Supported commands: `quit`, `filter all|t1|t2|t3`, `sort time|tier|source`.
- `Esc` cancels command mode without executing.
- Unknown command: display `Unknown command: {input}` inline for 2 seconds then clear.

### Help Overlay (`?`)

- Full-screen overlay showing all key bindings and command vocabulary.
- Any key dismisses the overlay.

## Component Inventory (Ratatui Widgets)

| Widget | Ratatui Type | Usage |
|--------|-------------|-------|
| Stats header | `Block` + `Paragraph` | Top stats bar with live counters |
| Filter bar | `Block` + `Paragraph` or `Tabs` | Filter/sort state display |
| Event table | `Table` with `TableState` | Main scrollable event list |
| Key hint bar | `Paragraph` | Bottom 1-line hint, no border |
| Help overlay | `Block` + `Paragraph` centered | Key binding reference |
| Command input | `Paragraph` + cursor | `:` command line at bottom |
| New events indicator | inline `Span` in filter bar | "N new" badge when scrolled up |

All widgets rendered via `crossterm` backend. No alternate screen persistence —
cleanup restores terminal on exit (normal or panic).

---

## Accessibility and Terminal Compatibility

- All semantic information conveyed by both color AND text/symbol. No
  color-only differentiation (replay flag uses " [R] " text, not just red color).
- ANSI color codes gated behind `crossterm::style::Colors` feature detection.
  Graceful degradation to 16-color ANSI on terminals that do not support 256/RGB.
- Mouse support: not required for v1. Keyboard-only navigation sufficient.
- Minimum terminal: 80×20. Below minimum: render a single-line message
  `Terminal too small (min 80×20). Resize to continue.`
