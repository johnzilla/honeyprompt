**src/nonce.rs** — Cryptographic nonce generation per D-11, GEN-03, Pattern 5 from research:

```rust
/// Generate a 16-character lowercase hex nonce from 8 bytes of OS CSPRNG.
pub fn generate_nonce() -> String {
    let mut buf = [0u8; 8];
    getrandom::fill(&mut buf).expect("OS CSPRNG unavailable");
    hex::encode(buf)
}
```

**src/store/mod.rs** — SQLite schema per SRV-02, Pattern 6 from research. Per Pitfall 3: ALL queries use `params![]`, NEVER `format!()`.

```rust
use rusqlite::{Connection, params};
```

Implement:
- `pub fn open_or_create_db(path: &std::path::Path) -> anyhow::Result<Connection>` — opens SQLite at path, calls `run_migrations`, returns connection
- `pub fn run_migrations(conn: &Connection) -> rusqlite::Result<()>` — executes the schema DDL below
- `pub fn insert_nonce(conn: &Connection, nonce: &str, tier: u8, payload_id: &str, embedding_loc: &str) -> rusqlite::Result<()>` — inserts into `nonce_map` table using `params![]`

Schema DDL (execute via `conn.execute_batch`):
```sql
PRAGMA journal_mode = WAL;

CREATE TABLE IF NOT EXISTS events (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    nonce           TEXT NOT NULL,
    tier            INTEGER NOT NULL,
    payload_id      TEXT NOT NULL,
    embedding_loc   TEXT NOT NULL,
    first_seen_at   TEXT NOT NULL,
    last_seen_at    TEXT NOT NULL,
    fire_count      INTEGER NOT NULL DEFAULT 1,
    is_replay       INTEGER NOT NULL DEFAULT 0,
    session_id      TEXT,
    remote_addr     TEXT,
    user_agent      TEXT,
    extra_headers   TEXT
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_events_nonce ON events(nonce);

CREATE TABLE IF NOT EXISTS nonce_map (
    nonce           TEXT PRIMARY KEY,
    tier            INTEGER NOT NULL,
    payload_id      TEXT NOT NULL,
    embedding_loc   TEXT NOT NULL,
    generated_at    TEXT NOT NULL
);
```

The `events` table has replay detection fields (`fire_count`, `is_replay`, `first_seen_at`, `last_seen_at`) and session grouping (`session_id`) per SRV-02. The `session_id` column is nullable TEXT — Phase 2 server code populates it.

- `test_parameterized_insert` — insert a nonce containing SQL metacharacters (`'; DROP TABLE nonce_map; --`), verify DB is not corrupted and the nonce is stored literally (Pitfall 3 mitigation)
