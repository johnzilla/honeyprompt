Create `deploy/README-deploy.md` per D-07. This is the complete manual deployment guide. Structure:

**Header:** "Deploying HoneyPrompt to DigitalOcean" with a brief overview: static musl binary + systemd + Caddy TLS on a $4-6/month droplet.

**Prerequisites section:**
- DigitalOcean account with a provisioned Ubuntu 22.04+ droplet (1 GB RAM, 25 GB disk is sufficient)
- SSH access to the droplet
- DNS A record for `honeyprompt.sh` pointing to the droplet's public IP
- A GitHub Release binary available (from Phase 6 release workflow)

**Section 1: Initial Server Setup**
- Create dedicated system user: `sudo useradd --system --no-create-home --shell /usr/sbin/nologin honeyprompt`
- Create data directory: `sudo mkdir -p /var/lib/honeyprompt` and `sudo chown honeyprompt:honeyprompt /var/lib/honeyprompt`
- Copy landing directory from repo to droplet: `sudo mkdir -p /var/lib/honeyprompt/landing` then rsync or scp `landing/` contents (honeyprompt.toml + output/) to `/var/lib/honeyprompt/landing/`
- Verify file structure: list expected paths (`/var/lib/honeyprompt/landing/honeyprompt.toml`, `/var/lib/honeyprompt/landing/output/index.html`, `/var/lib/honeyprompt/landing/output/callback-map.json`)

**Section 2: Install Binary**
- Download from GitHub Releases: `curl -L https://github.com/johnzilla/honeyprompt/releases/latest/download/honeyprompt-x86_64-unknown-linux-musl.tar.gz | tar -xz -C /tmp/`
- Install: `sudo mv /tmp/honeyprompt /usr/local/bin/honeyprompt && sudo chmod +x /usr/local/bin/honeyprompt`
- Verify: `/usr/local/bin/honeyprompt --version`

**Section 3: Configure systemd**
- Copy unit file: `sudo cp deploy/honeyprompt.service /etc/systemd/system/honeyprompt.service`
- Reload systemd: `sudo systemctl daemon-reload`
- Enable on boot: `sudo systemctl enable honeyprompt`
- Start: `sudo systemctl start honeyprompt`
- Verify: `sudo systemctl status honeyprompt` (should show Active: running), `journalctl -u honeyprompt -n 20` (should show "honeyprompt serve" startup with bind address and nonce count)

**Section 4: Install and Configure Caddy**
- Install Caddy via official APT repo (exact commands from research):
  ```
  sudo apt-get install -y debian-keyring debian-archive-keyring apt-transport-https curl
  curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
  curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
  sudo apt-get update && sudo apt-get install caddy
  ```
- Copy Caddyfile: `sudo cp deploy/Caddyfile /etc/caddy/Caddyfile`
- IMPORTANT: Before starting Caddy, verify DNS has propagated: `dig +short honeyprompt.sh @1.1.1.1` must return the droplet IP (Pitfall 1)
- Reload Caddy: `sudo systemctl reload caddy`
- Watch Caddy logs for cert provisioning: `journalctl -u caddy -f` — wait for successful ACME challenge
- Verify HTTPS: `curl -I https://honeyprompt.sh` should return 200

**Section 5: Firewall**
- Configure DigitalOcean Cloud Firewall or ufw to allow ONLY ports 22 (SSH), 80 (HTTP/ACME), 443 (HTTPS)
- Port 8080 must NOT be publicly accessible (Pitfall 3)
- Verify: `curl http://droplet-ip:8080` from an external machine should timeout/refuse

**Section 6: Uptime Monitoring (DEPLOY-03)**
- Sign up for UptimeRobot free tier at https://uptimerobot.com
- Create HTTPS monitor for `https://honeyprompt.sh` with 5-minute polling interval
- Configure email alert contact
- Note: UptimeRobot checks from external locations — detects network failures that on-server checks cannot

**Section 7: Binary Upgrade Procedure**
- Pre-upgrade backup: `sudo cp /var/lib/honeyprompt/landing/.honeyprompt/events.db ~/events.db.$(date +%Y%m%d)`
- Download new binary, stop service, replace binary, start service (exact commands)
- CRITICAL: The SQLite DB at `/var/lib/honeyprompt/landing/.honeyprompt/events.db` is the evidence store. It is NEVER touched during upgrades. Only `/usr/local/bin/honeyprompt` is replaced.

**Section 8: Troubleshooting**
- Service cycling (auto-restart loop): check `journalctl -u honeyprompt -n 50` for missing files or permission errors
- Caddy TLS failure: check DNS propagation, check Caddy logs, retry with `sudo systemctl restart caddy`
- DB permission denied: verify `sudo -u honeyprompt touch /var/lib/honeyprompt/landing/.honeyprompt/test`

**Section 9: Architecture Reference**
- ASCII diagram of the deployment: Internet -> Caddy (:443 TLS) -> honeyprompt (:8080) -> SQLite DB
- File layout on the droplet (from research: recommended directory layout)
