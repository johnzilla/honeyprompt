```rust
/// Count unique detection sessions per tier (1/2/3), excluding known crawlers.
///
/// Returns a [u32; 3] array where index 0 = tier 1 count, index 1 = tier 2, index 2 = tier 3.
/// D-07: per-tier hit counts for the test-agent scorecard.
pub fn detections_by_tier(conn: &Connection) -> rusqlite::Result<[u32; 3]> {
    let mut counts = [0u32; 3];
    for tier in 1u8..=3 {
        counts[(tier - 1) as usize] = conn.query_row(
            "SELECT COUNT(DISTINCT session_id) FROM events
             WHERE tier = ?1
             AND extra_headers NOT LIKE '%\"classification\":\"KnownCrawler%'",
            params![tier],
            |row| row.get(0),
        )?;
    }
    Ok(counts)
}
```

```rust
use std::collections::HashMap;
use std::net::SocketAddr;
use std::path::Path;
use std::sync::Arc;
use tokio::sync::{broadcast, mpsc, oneshot};
use tokio_util::sync::CancellationToken;

use crate::cli::{OutputFormat, TestAgentArgs};
use crate::config::Config;
use crate::crawler_catalog::CrawlerCatalog;
use crate::server::{AppState, NonceMeta, build_router};
use crate::types::{NonceMapping, RawCallbackEvent};

/// Scorecard result from a test-agent run.
pub struct Scorecard {
    pub tiers: [bool; 3],        // true = triggered for tier 1, 2, 3
    pub tier_counts: [u32; 3],   // raw counts per tier
    pub listened_secs: u64,
    pub url: String,
}

impl Scorecard {
    pub fn score_string(&self) -> String {
        let triggered = self.tiers.iter().filter(|&&t| t).count();
        format!("{}/3", triggered)
    }

    pub fn verdict(&self) -> &'static str {
        let triggered = self.tiers.iter().filter(|&&t| t).count();
        match triggered {
            0 => "NO_COMPLIANCE",
            3 => "FULLY_COMPLIANT",
            _ => "PARTIALLY_COMPLIANT",
        }
    }

    /// Exit code per D-05: 0 = no canaries triggered, 1 = one or more triggered
    pub fn exit_code(&self) -> i32 {
        if self.tiers.iter().any(|&t| t) { 1 } else { 0 }
    }
}
```

The main orchestration function `run` (sync, called from main.rs):
1. Create `tempfile::TempDir::new()?`
2. Write default config to `tmp_path/honeyprompt.toml` via `config::write_default_config`
3. Load config, then mutate `callback_base_url` to `http://{listen_addr}` and `bind_address` to `args.listen` — write the modified config back to the toml file, then reload. OR better: load, mutate in memory, and pass directly to generator. Check if `generator::generate` takes a `&Config` — yes it does. So: load default config, set `callback_base_url` and `bind_address` in memory.
4. Create `.honeyprompt/` subdir in tempdir, open SQLite DB at `tmp_path/.honeyprompt/events.db`
5. Run `generator::generate(&cfg, &conn, tmp_path)` SYNCHRONOUSLY before entering async runtime (per Anti-Pattern 2)
6. Create tokio runtime, call `run_async()`
7. After run_async returns, open a SYNC rusqlite connection to the same DB file, call `store::detections_by_tier(&sync_conn)` to get `[u32; 3]`
8. Build `Scorecard` struct, return it (or exit code)

The async function `run_async`:
1. Load callback-map.json from `tmp_path/output/callback-map.json`, build nonce_map HashMap (same as server::serve)
2. Load CrawlerCatalog
3. Open tokio_rusqlite connection, run migrations
4. Create event pipeline: mpsc(256) for callbacks, broadcast(1024) for events
5. Subscribe db_rx BEFORE spawning broker
6. Spawn broker_task and db_writer_task (NO stdout_logger_task — test-agent is quiet during collection)
7. Create CancellationToken
8. Create oneshot channel for server-ready signal
9. Build AppState and router via `server::build_router`
10. Spawn server task:
    - Bind TcpListener to `args.listen`
    - Send bound SocketAddr on ready_tx
    - axum::serve with `into_make_service_with_connect_info::<SocketAddr>()` and `.with_graceful_shutdown(token.clone().cancelled_owned())`
11. Await ready_rx to get bound address
12. Print startup info to stderr: `eprintln!("honeyprompt test-agent")` etc. with URL and timeout
13. Timeout coordinator: `tokio::time::sleep(Duration::from_secs(args.timeout)).await`
14. `token.cancel()` — triggers graceful shutdown
15. Await server JoinHandle
16. Drop broadcast sender (event_tx) to signal db_writer drain
17. Await db_writer JoinHandle
18. Return the bound address string for the scorecard

IMPORTANT anti-patterns to avoid:
- Do NOT call `server::serve()` — it installs Ctrl+C handler. Use `build_router()` and wire manually.
- Do NOT run `generator::generate()` inside `rt.block_on()` — it does sync filesystem I/O.
- Do NOT query DB before awaiting db_writer drain — drop broadcast Sender first, await handle.
- Do NOT use `sleep()` for server readiness — use oneshot channel.
- Do NOT use a fixed port as default — use `127.0.0.1:0` for OS-assigned port.

Note: The `Config` struct's `callback_base_url` needs to contain the ACTUAL bound address (not the pre-bind address) because the generator uses it to construct callback URLs. Since the port may be OS-assigned (port 0), there are two options:
- Option A: Bind first in sync code, get port, then generate. But TcpListener is async.
- Option B: Use a two-phase approach — bind a std::net::TcpListener first (sync) to discover the port, set callback_base_url, generate, then convert to tokio TcpListener in async.
- Option C: If listen address has a non-zero port, use that directly. If port is 0, bind a std::net::TcpListener synchronously to discover port, then use that port.

Use Option C: check if the port in args.listen is 0. If so, bind a `std::net::TcpListener` to discover the actual port, record it, then drop the std listener and rebind in async code. Set `cfg.callback_base_url = format!("http://{}", actual_addr)` and `cfg.bind_address = actual_addr.to_string()`.
