<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Catalog TOML → generated HTML | Catalog is embedded + curated; generator trusts its contents. Risk only if catalog authoring goes wrong (mitigated by Plan 01's test_tier5_catalog check). |
| Generated nonce → derive_seed | `generate_nonce` always returns 16-char hex, so `derive_seed.expect(...)` is sound as a generator invariant. An invariant-break panics the generator at build time (loud failure — preferable to silent wrong-seed emission). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Info Disclosure | seed JSON-LD block | mitigate | Seed is derived from the public nonce — no secret material is transmitted through the seed. The nonce itself is public in the URL; including it in the JSON-LD block lets agents disambiguate when multiple T5 payloads coexist, without leaking any non-public information. Recorded as Q1 resolution in RESEARCH.md lines 784-788. |
| T-13-PLACEHOLDER-LEAK | Info Disclosure | Unresolved placeholder in rendered HTML | mitigate | Both `test_tier4_renders_with_b64_base` and `test_tier5_renders_with_proof_base` assert no raw `{callback_url_*_base}` text survives in the output. Leaking an unresolved placeholder would (a) look broken, (b) tip off the agent to a templating system. |

Note on T-13-03 (/cb/v1/ byte-identity regression): This plan makes NO changes to the Tier1/Tier2/Tier3 match arms. `test_generate.rs` existing tests enforce this — D-13-18 regression guard is automatic.
