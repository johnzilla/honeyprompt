**src/generator/mod.rs** — Template rendering pipeline using minijinja and rust-embed:

```rust
use rust_embed::RustEmbed;
use minijinja::{Environment, context};
```

Create a `#[derive(RustEmbed)] #[folder = "assets/templates/"] struct Templates;`

Implement:
- `pub fn generate(config: &Config, conn: &Connection) -> anyhow::Result<()>` — the top-level generate function:
  1. Load payloads via `catalog::load_for_tiers(&config.tiers)`
  2. For each payload, call `nonce::generate_nonce()` to get a nonce
  3. For Tier 1 payloads: create callback URL as `{callback_base_url}/cb/{nonce}`, substitute `{callback_url}` in instruction
  4. For Tier 2 payloads: generate TWO nonces, create `callback_url_a` and `callback_url_b`, substitute both in instruction
  5. For Tier 3 payloads: create `callback_url_base` as `{callback_base_url}/cb/{nonce}`, substitute `{callback_url_base}` in instruction
  6. Call `store::insert_nonce` for each nonce generated (per D-12)
  7. Render index.html template with payloads context (each payload gets `rendered_instruction` field with nonce-substituted text)
  8. Render robots.txt template
  9. Render ai.txt template with `callback_base_url` and `page_title` context
  10. Build callback-map.json as `Vec<NonceMapping>` serialized with `serde_json::to_string_pretty`
  11. Write all 4 files to `{path}/output/` directory (create if needed)

- Helper: `fn render_template(name: &str, ctx: minijinja::Value) -> anyhow::Result<String>` — loads template from embedded assets, renders with minijinja Environment

Build a `RenderedPayload` struct for template context:
```rust
struct RenderedPayload {
    embedding_location: String,
    rendered_instruction: String,
}
```

Per D-07: output goes to `output/` containing index.html, robots.txt, ai.txt, callback-map.json.
